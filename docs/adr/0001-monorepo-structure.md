# ADR 0001: モノレポ構成を採用する

- ステータス: 採用
- 日付: 2026-07-07

## コンテキスト

現フェーズはWITインターフェース（`mycflow:dataflow` / `mycflow:node`）の設計中であり、Docker初期環境をターゲットにしている。
WIT定義の変更はホスト側実装（`host-dataflow` / `placement-manager` / `node-agent`）とWASMコンポーネント（`components/`）の双方に波及する。

## 決定

`wit/` `crates/` `components/` `compositions/` `deploy/` `config/` `docs/` を単一リポジトリ（本リポジトリ）にまとめるモノレポ構成を採用する。

## 理由

インターフェースが未凍結の段階では、破壊的変更をホスト側実装とコンポーネント側で同一コミット内にアトミックに反映できることを最優先する。マルチリポジトリでのバージョン同期コストは、この段階では設計速度に対するコストの方が大きい。

## 結果

- `wit/` はOCIレジストリへ`wkg publish`する一次情報として独立ディレクトリに置き、`crates/`配下には埋め込まない。
- インターフェースが凍結された後、`wit/`は独立リポジトリ+バージョニングへ切り出す候補。
- 組み込みファームウェア（Zephyr/ESP-IDF）のビルドシステムはCargoワークスペースと共存が難しいため、実機フェーズで`mycflow-firmware`として分離する（現時点ではDockerスコープのため対象外）。
